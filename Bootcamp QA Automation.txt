Bootcamp QA Automation
DAY 1
Planning: PM akan presentasi product yang akan di develope, qa  berperan sebagai requrement analysis untuk deteksi bug sebelum masuk proses development
Smoke test: menguji aplikasi sebelum rilis untuk fitur yang utama/major
sanity tes: menguji fitur untuk improvement ketika existing apps yang baru naik/telah ke production
setelah nya bikin qa sign off (laporan testcase)
regression test: test eksekusi dari test case
rilis train: menguji fitur yang sudah didevelope tapi belum diaktifkan
monolitic test: pengujian langsung di frontend apps
entire test: pengujian pemisahan frontend dan backend (unit test -> backend test/service test/api test/integrasi test -> ui test user experience, and repeat)

mindset qa automation test:
sebelum automation test, harus berdiri dulu sebagai manual test, harus ada test case manual yang baik dulu, 
automation adalah alat untuk mempermudah kita dengan strategi test case, automasi test case yang berulang-ulang, perlu keseuaian dengan habbit user dengan hasil yang sama setiap saat, 
kenapa automation? regresi lebih cepat, test otomatis jalan setiap commit code (ci/cd ready), data driven/test case/input banyak bisa lebih fleksibel/efektif, test yang butuh kepresisian perhitungan, test yang butuh banyak browser/device.
automation adalah invest jangka panjang.

automation tools compare:
selenium: release paling awal (2004), speed lebih lambat, multi browser, manual wait, pararel eksekusi setup, typescript per;u setup, recording tool ide/katalon

cypress: release 2017, speed cepat, chrome family, built-in, prarel eksekusi built-in, type script built in, recording tool test runner

playwright: release 2020, speed tercepat, all web, smart wait, pararel eksekusi default/mudah, type script first class, recording tool codegen (best)
   
Playwright setup and demo:1. Buat folder D:\bootcamp_qa, 2. install nodejs di folder ini, 3.install playwright D:\bootcamp_qa>npm init playwright, 4. bikin script: npx playwright codegen --> untuk record web test --> auto buat script --> Copy script --> buat file namafile.spec.ts pada folder /tests --> paste di code editor
jalankan test dengan command=npx playwright test tests/namafile.spec.ts --headed (--headed untuk menampilkan proses testing) --> untuk lihat hasil test=npx playwright show-report
